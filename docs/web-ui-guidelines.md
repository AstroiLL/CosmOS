# Web UI Guidelines

## Источники визуального направления

Папка `img/` содержит примеры желаемого веб-интерфейса. Перед проектированием Web UI агентам нужно смотреть эти изображения и использовать их как референсы по стилю, плотности интерфейса и общему ощущению продукта.

## Принципы интерфейса

- **Современный** — аккуратный, чистый, без устаревших UI-паттернов.
- **Простой** — минимум визуального шума, понятные основные действия.
- **Легко редактируемый агентами** — структура компонентов и стилей должна быть очевидной, без чрезмерной магии.
- **Модульный** — отдельные экраны, панели и карточки должны быть независимыми компонентами.
- **Текстовые конфиги там, где возможно** — навигация, агенты, memory backends и команды должны описываться декларативно.

## Практические требования

- Использовать понятные имена файлов и компонентов.
- Держать стили рядом с компонентами или в простой дизайн-системе.
- Избегать сложных CSS-трюков, если их трудно поддерживать агентам.
- Предпочитать небольшие компоненты вместо больших монолитных страниц.
- Документировать нестандартные решения прямо в коде или рядом в Markdown.

## Цель

Web-интерфейс CosmOS должен ощущаться как рабочая панель агентной ОС: простой вход в задачи, агентов, память, автоматизации и историю выполнения.
