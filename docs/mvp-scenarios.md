# MVP-сценарии CosmOS

## Сценарий 1: Простая задача через CLI

```
$ cosmos task "Найди последние новости про GRPO и сделай краткую выжимку"
```

CosmOS:
1. Создаёт задачу в SQLite.
2. Router выбирает Hermes (исследование + web).
3. Запускает `hermes chat -q "..."`.
4. Сохраняет результат и лог.
5. Выводит ответ в терминал.

---

## Сценарий 2: Кодовая задача через CLI

```
$ cosmos task "Добавь rate limiting в FastAPI приложение в ./myapp" --agent claude
```

CosmOS:
1. Создаёт задачу.
2. Явно указан агент — Claude Code.
3. Создаёт git worktree для изоляции.
4. Запускает `claude --print "..."`.
5. Проверяет diff, запускает тесты.
6. Сохраняет результат и артефакты.

---

## Сценарий 3: Статус задач

```
$ cosmos status
```

CosmOS выводит таблицу:
- ID задачи;
- описание;
- агент;
- статус (pending/running/completed/failed);
- время начала/окончания;
- краткий результат.

---

## Сценарий 4: Проверка окружения

```
$ cosmos doctor
```

CosmOS проверяет:
- Python версия;
- установленные агенты (hermes, claude, codex);
- доступность SQLite;
- путь к Obsidian vault;
- конфиг `cosmos.yaml`;
- переменные окружения из `.env`.

---

## Сценарий 5: Сохранение в память

После выполнения задачи CosmOS:
- пишет лог в SQLite;
- при необходимости создаёт заметку в Obsidian (в папку CosmOS);
- обновляет skills, если найден новый workflow.

---

## Сценарий 6: Перенос на другую машину

```
# На старой машине:
$ cosmos backup --output cosmos-backup.tar.gz

# На новой машине:
$ ./scripts/install.sh
$ cosmos restore cosmos-backup.tar.gz
$ cosmos doctor
```

CosmOS:
1. Разворачивает код и зависимости.
2. Восстанавливает SQLite state.
3. Восстанавливает конфиги.
4. Проверяет окружение.
5. Показывает, какие агенты/интеграции доступны, какие — нет.

---

## Сценарий 7: Задача через Telegram (Phase 4)

Пользователь пишет в Telegram:
> cosmos: найди инфу про MCP protocol v2

CosmOS:
1. Принимает задачу через Telegram gateway.
2. Создаёт задачу в SQLite.
3. Запускает Hermes.
4. Присылает результат в Telegram.

---

## Сценарий 8: Web UI (Phase 5)

Пользователь открывает Web UI:
- видит дашборд с активными задачами;
- создаёт новую задачу через форму;
- выбирает агента из выпадающего списка;
- видит историю выполнения;
- может отменить задачу.
